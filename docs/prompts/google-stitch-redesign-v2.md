# Google Stitch Redesign Prompt V2

Use this prompt after Stitch has already generated the first version and the result feels too much like a traditional admin dashboard.

```text
请重新设计当前所有页面。保留当前产品功能和中文内容，但整体视觉需要明显升级：现在的版本太像传统后台系统/通用深色数据大屏，缺少 Stock Opportunity Lab 自己的产品气质。

目标气质：
把它做成一个“高端个人量化投研终端”，像专业投研工作台，而不是 SaaS 后台。它应该数据密集、克制、漂亮、有辨识度。用户一眼能感觉这是一个用于 A 股盘后复盘、策略验证、消息归因和板块情绪判断的专业工具。

核心改造方向：
1. 不要使用传统后台模板：
   - 不要满屏均匀卡片网格。
   - 不要大面积深色 dashboard。
   - 不要所有模块都长得一样。
   - 不要 generic admin sidebar + generic KPI cards + generic table 的组合。

2. 采用更有产品记忆点的信息结构：
   - 顶部做“Market Regime Ribbon 市场状态带”：交易日、情绪状态、市场温度、上涨/下跌家数、成交额、数据源状态、策略版本。
   - 主区域做“Opportunity Command Center 机会中枢”：候选股票不是普通后台表格，而是带有排名、分数、价格计划、风险边界和板块确认状态的高密度投研列表。
   - 右侧做“Evidence Rail 证据侧栏”：消息异动、板块资金、风险提示、报告输出。
   - 详情页用“Thesis Sheet 投研卡片”结构：量价证据、价格计划、消息时间线、板块共振、历史相似样本。
   - 回测页用“Strategy Lab 策略实验室”结构：权益曲线、回撤曲线、收益分布、月度热力图、归因拆解和交易明细。

3. 视觉语言：
   - 主背景使用冷白/浅灰工作台，不要全黑。
   - 左侧导航可使用深墨蓝/炭黑，但内容区保持明亮。
   - 使用精致的细边框、分割线、浅色数据面板、轻微阴影。
   - 色彩克制但有层次：墨蓝、青绿色、琥珀色、风险红、少量电蓝作为操作色。
   - 数字必须使用 tabular numbers，价格/百分比/成交额要对齐。
   - 卡片圆角不超过 8px。
   - 不使用大面积渐变、玻璃拟态、股票照片、营销插画。

4. 数据可视化要更有金融产品质感：
   - 使用细线 sparkline、迷你 K 线、资金流热力图、情绪温度条、分布直方图。
   - 板块资金页面不要只是表格，要有 sector flow map / heatmap。
   - 消息异动页面要像事件调查台：左侧异动列表，右侧事件时间线和机会/风险归因。
   - 回测实验室要有“实验感”：策略版本、样本区间、收益曲线、回撤、分桶归因、失败样本。

5. 交互层级：
   - Dashboard 首屏先回答：今天市场适不适合做？哪些板块强？哪些股票进入观察？有什么消息风险？
   - 所有候选都必须显示：计划低吸价、计划买入上限、高开放弃价、止损参考、机会标签、风险条件。
   - 所有消息分析都要显示来源/时间/置信度；没有来源时显示“消息源不足”，不要编造。
   - 所有推荐都要带“不构成投资建议”的风险语义。

请重新生成 6 个页面：
1. 今日机会 Dashboard
2. 个股机会详情 - 华宏科技
3. 回测实验室
4. 消息异动
5. 板块资金
6. 策略设置

每个页面都要使用统一设计语言，但布局不要完全一样。请让页面更像一个漂亮、专业、真实可用的金融研究产品，而不是传统后台管理系统。
```
